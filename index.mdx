Adsum IoT Coder is an AI coding agent for VS Code that handles your entire embedded firmware workflow on **Espressif ESP** and **Nordic nRF** platforms. Unlike general coding assistants, it reads your board — capturing live logs, correlating multi-device state, and drawing on human-curated firmware expertise — so it can close bugs that only show up at runtime.

## What ships today

Adsum IoT Coder supports the full inner loop across two major IoT platforms:

- **Nordic nRF52 / nRF53 / nRF54** on nRF Connect SDK (Zephyr) — BLE
- **Espressif ESP32, ESP32-S3, ESP32-C6** on ESP-IDF — BLE and Wi-Fi

Four built-in workflows cover the daily work: **prototype**, **build/flash/debug**, **add a feature**, and **test & validate**. The agent suggests the next move after every task, so you are never left at a blank prompt.

## Get up and running

<Steps>
  <Step title="Install the extension">
    Open the VS Code **Extensions** panel, search for **Adsum IoT Coder**, and click Install. Or go directly to the [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=AdsumNetwork.nrf-ai-debugger).
  </Step>
  <Step title="Run the built-in demo">
    The home screen opens a live demo that debugs a real BLE bug on bundled firmware — no board or project required. This is the fastest way to see how the agent works.
  </Step>
  <Step title="Open your project">
    Open your nRF or ESP project folder. The agent detects your platform, boards, and toolchain automatically, then surfaces the right one-click actions for your setup.
  </Step>
  <Step title="Add your own key (optional)">
    The [free tier](/free-tier) is on by default and requires no setup. When you want a specific model or heavier usage, drop in any OpenAI-compatible key in the [Models settings](/models).
  </Step>
</Steps>
