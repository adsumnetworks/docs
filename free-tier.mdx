Most tools ask you to choose a provider, paste an API key, and add a payment card before you can find out whether they actually help. Adsum IoT Coder skips all of that.

Install the extension and it just works. Inference runs on a managed model hosted by Adsum Networks, so you can point the agent at your own firmware in the first minute, not the first hour. The free tier is a real working tier — generous enough to scaffold a project and run a complete debug loop — not a locked demo with a nag screen.

<Note>
  No key, no account, no card. The free tier is on by default.
</Note>

## Free tier vs bring your own key

When you are ready for heavier usage, a specific model, or longer sessions, you can switch to your own key at any time. The comparison below covers the key differences.

| | Free Tier | Bring Your Own Key |
|:---|:---|:---|
| **API key required** | No | Yes |
| **Cost to you** | Nothing — inference is on Adsum | Your provider's standard rates |
| **Model** | Managed by Adsum Networks | Any OpenAI-compatible model or endpoint |
| **Best for** | First run, evaluation, quick fixes | Daily use, long sessions, model choice |

## How to switch to your own key

When you want a specific model or need heavier usage, drop in any OpenAI-compatible API key. The switch is instant: the task you are currently running keeps going without a restart, and all future tasks use your key.

See [Models](/models) for the list of supported providers, recommended model settings, and the exact configuration used in the benchmark.

## The token meter

The free tier is token-metered. A **tokens left** indicator in the extension decrements with real usage as you work. Your balance is tracked and enforced by Adsum Networks. When you reach the limit, a one-click prompt moves you onto your own key and the same task picks up exactly where it left off — no work is lost.
